# so can be ignored, use TokenCreature instead)

# Use verify test to check it: test_checkMissingTokenData

# Inner/xmage related tokens stores in TokenRepository (copy, morph, etc)

# - use simple name for the emblem like Gideon
# - all names must be different in one set
# - xmage do not support various arts for the emblems (e.g. must be one emblem's image per set)
# - download images must be setup in ScryfallImageSupportTokens

# Schema:
# Currently defines 4 columns:
# - 1. Type and set. In "TYPE:SETCODE" format
#   - Valid values for TYPE include: EMBLEM, PLANE, DUNGEON, TOK
#   - SETCODE maps to MTG set codes
# - 2. Token Name
# - 3. Image Number. Used if one set contains multiple tokens with same name. Nullable.
# - 4. Token Class Name. Used for image search for rendering.
